# Генератор линейных git-репозиториев для практики bisect

## Идея

Сыграйте с Git в игру «угадай число»: найдите коммит, в котором нужное число было добавлено в файл (числа не повторяются).

- Диапазон чисел: от 1 до N, где N — число коммитов в репозитории.
- В каждом коммите добавляется одно число из диапазона, выбранное случайным образом и вставленное в случайное место файла.
- В части коммитов, помимо добавления, до 4 уже существующих строк переносятся в произвольные позиции файла (не обязательно рядом с исходным местом). Вероятности заданы в коде (`_RELOCATE_LINE_COUNT_WEIGHTS` в `bisect_repo_gen.py`).
- Часть коммитов специально «испорчена»: числа склеены без переносов строк, как в трудночитаемых или «битых» коммитах. При желании добавленное число всё равно можно найти.
- По умолчанию в начале `file.txt` есть текст-подсказка («нормальный» / «сломанный» коммит); отключить: `--no-commit-hints`.

## Требования

- Python 3
- `git` в PATH (доступен из консоли как `git`)

## Использование

По умолчанию создаётся репозиторий `repos/practice100` со 100 коммитами:

```bash
python bisect_repo_gen.py
```

Другой размер и имя:

```bash
python bisect_repo_gen.py practice300 -n 300
```

Справка по аргументам:

```bash
python bisect_repo_gen.py --help
```

| Аргумент | По умолчанию | Описание |
|----------|--------------|----------|
| `repo_name` | `practice100` | Имя папки репозитория внутри `--output-dir` |
| `-n`, `--commit-count` | `100` | Число коммитов (= диапазон чисел 1..N) |
| `-f`, `--target-file` | `file.txt` | Имя файла в репозитории |
| `--broken-chance` | `0.1` | Вероятность «сломанного» коммита (0..1) |
| `--commit-hints` / `--no-commit-hints` | включены | Заголовок-подсказка в `file.txt` |
| `-o`, `--output-dir` | `repos` | Каталог для репозиториев |
| `--seed` | — | Seed для воспроизводимой генерации |
| `--force` | — | Перезаписать существующий каталог |

Пример с фиксированным seed:

```bash
python bisect_repo_gen.py test_repo -n 20 --seed 42
```

## Настройка

- Больше «испорченных» коммитов: увеличьте `--broken-chance` (например, `0.25`).
- Без подсказок в файле (сложнее bisect): `--no-commit-hints`.
- Воспроизводимая последовательность: задайте `--seed`.

## Практика git bisect

```bash
cd repos/practice300
git bisect start
git bisect bad HEAD
git bisect good <hash-первого-коммита>
# на каждом шаге смотрите file.txt и отмечайте good/bad
git bisect reset
```
